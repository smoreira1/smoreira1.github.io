Markdown

El contenido de Markdown se usa comúnmente para crear contenido con mucho texto, como artículos de blog y documentación. Astro incluye soporte integrado para markdown con algunas características adicionales como soporte para expresiones JavaScript y componentes de Astro.

Páginas de Markdown

Section titled Páginas de Markdown

Astro trata cualquier archivo .md dentro de la carpeta /src/pages como una página. Al colocar un archivo en esta carpeta, o en cualquier subcarpeta, se creará automáticamente una ruta de página utilizando la ruta del archivo.

📚 Obtenga más información sobre enrutamiento basado en archivos en Astro.

Ejemplo básico

Section titled Ejemplo básico

La forma más fácil de comenzar a usar Markdown en Astro es crear una ruta de la página de inicio src/pages/index.md en su proyecto. Copie el código a continuación en su proyecto y luego podrá ver el HTML renderizado en la página de inicio de su proyecto. Por lo general, se encuentra en http://localhost:3000/.

Plantillas de Markdown

Section titled Plantillas de Markdown

Las páginas de Markdown tienen una propiedad frontmatter especial para layout que define la ruta relativa a un componente plantilla de Astro. Este componente envolverá su contenido Markdown, proporcionando una capa contenedora y cualquier otro elemento incluido en la plantilla de la página.

Una plantilla común para páginas de Markdown incluye:

  1. la propiedad content para acceder a los metadatos de la página de Markdown.
  2. un <slot /> predeterminado para indicar dónde debe mostrarse el contenido de la página de Markdown.

La propiedad content también contiene una propiedad astro con metadatos adicionales sobre la página, como el texto Markdown fuente y un objeto headers.

Frontmatter como props

Section titled Frontmatter como props

Cualquier componente de Astro (¡no solo las plantillas!) pueden recibir valores definidos en el frontmatter del Markdown como props. También puedes especificar varios tipos de datos usando YAML en el frontmatter y capturar metadatos de cada publicación de blog para usar en su proyecto de Astro.

Acceda a estos valores en cualquier archivo .astro como lo haría en un componente plantilla, como se describe anteriormente.

IDs de título

Section titled IDs de título

Astro agregará identificaciones generadas automáticamente a todos los títulos y subtítulos en los archivos Markdown usando github-slugger. Pero, si se especifica una identificación personalizada, no será sobrescrita.

Estas identificaciones se agregarán después de que se ejecuten todos los demás complementos, por lo que si tiene un complemento como rehype-toc que necesita identificaciones, deberá agregar su propio plugin de slug (como rehype-slug).

Borradores en Markdown

Section titled Borradores en Markdown

draft: true es un valor opcional que marcará una página o artículo individual .md como “borrador”. De forma predeterminada, esta página se excluirá de la compilación final de su proyecto.

Las páginas de Markdown sin la propiedad draft o aquellas con draft: false no se verán afectadas y se incluirán en la compilación final.

Borradores y Astro.glob()

Aunque draft: true evitará que se construya la página de su proyecto, este archivo estará disponible para Astro.glob() el cual devuelve todos los archivos Markdown en la ruta especificada.

Para evitar que los borradores sean incluidos en un registro de artículos o en la lista de artículos más recientes, asegúrese de filtrar el resultado devuelto por Astro.glob().

⚙️ Para habilitar la creación de páginas de borrador:

Agregue drafts: true a markdown en astro.config.mjs

Sugerencia

¡También puedes agregar la extensión --drafts al ejecutar astro build para incluir la creación de páginas borrador!

Markdown de Astro

Section titled Markdown de Astro

Deprecated

Astro v1.0 ya no admite componentes o JSX en las páginas de Markdown de forma predeterminada (EN) y es posible que se elimine en una versión futura.

Mientras tanto, la configuración de Astro admite una legacy flag que reactivará estas funcionalidades en páginas de Markdown hasta que pueda migrar a @astrojs/mdx (EN).

Propiedades exportadas

Section titled Propiedades exportadas

Cada archivo Markdown exporta las siguientes propiedades:

frontmatter

Section titled frontmatter

Contiene cualquier dato especificado en el frontmatter YAML de este archivo.

file

Section titled file

La ruta absoluta de este archivo (por ejemplo, /home/user/projects/.../file.md).

url

Section titled url

Si es una página, contiene la URL de la página (por ejemplo, /es/guides/markdown-content).

getHeaders()

Section titled getHeaders()

Una función asíncrona que devuelve los encabezados del archivo Markdown. La respuesta sigue este tipo: { depth: number; slug: string; text: string }[].

rawContent()

Section titled rawContent()

Una función que devuelve el contenido sin procesar del archivo Markdown (excluyendo el bloque frontmatter) como un string. Esto es útil cuando, por ejemplo, se calculan los “minutos leídos”. Este ejemplo usa el popular paquete reading-time:

compiledContent()

Section titled compiledContent()

Una función asíncrona que devuelve el contenido compilado a una sintaxis de Astro válida. Nota: ¡Esto no analiza {expresiones jsx}, <Componentes /> o componentes plantilla! Solo los bloques de Markdown estándar como ##títulos y -listas se compilarán a HTML. Esto es útil cuando, por ejemplo, tienes un bloque de resumen dentro del artículo de blog. Configuración de Markdown

Section titled Configuración de Markdown

El soporte de Markdown en Astro está basado en remark, una potente herramienta de análisis sintáctico y procesamiento con un ecosistema activo. Otros analizadores de Markdown como Pandoc y markdown-it no están actualmente soportados.

Puedes personalizar cómo remark analiza tu Markdown en astro.config.mjs. Consulte la documentación de referencia para más detalles de configuración o siga nuestras guías a continuación sobre cómo agregar plugins de remark y personalizar el resaltado de sintaxis.

Plugins de Markdown

Section titled Plugins de Markdown

Astro es compatible con plugins externos como remark y rehype. Puede proporcionar otros plugins en astro.config.mjs.

Nota

De forma predeterminada, Astro viene con GitHub flavored markdown y remark-smartypants habilitados.

Habilitar remarkPlugins o rehypePlugins personalizados eliminará estos complementos integrados y deberá agregarlos explícitamente si lo desea.

Resaltado de sintaxis

Section titled Resaltado de sintaxis

Astro viene con soporte integrado para Shiki y Prism. Esto proporciona un resaltado de sintaxis instantáneo para:

  • todas las vallas de código (```) usadas en los archivos markdown (.md).
  • el contenido dentro del componente <Code /> (con la tecnología de Shiki), o el componente <Prism /> (con la tecnología de Prism).

Shiki está habilitado de forma predeterminada, preconfigurado con el tema github-dark. La salida compilada se limitará a estilos en línea sin clases CSS externas, hojas de estilo o JS del lado del cliente.

Si opta por usar Prism, se aplicaran las clases CSS de Prism en su lugar. ¡Tenga en cuenta que necesita aportar su propia hoja de estilo CSS para que aparezca el resaltado de sintaxis! Consulte la sección de configuración de Prism para obtener más detalles.

Configuración de Prism

Section titled Configuración de Prism

Cuando use Prism, deberá agregar una hoja de estilos a su proyecto para resaltar la sintaxis. Si recién empieza y prefiere usar Prism en lugar de Shiki, le sugerimos:

  1. Configurar syntaxHighlight: 'prism' desde su configuración @astrojs/markdown-remark.
  2. Escoger una hoja de estilo prediseñada de los temas de Prism disponibles.
  3. Agregar esta hoja de estilo a la carpeta public/ de su proyecto.
  4. Cargar esta en el <head> de su página a través de una etiqueta <link>.

También puede visitar la lista de lenguajes de programación soportados por Prism para conocer las opciones y el uso.
